# 内容替换手册

本项目已拆分为**引擎 + 数据**两层：

| 文件 | 作用 | 换内容时要不要动 |
|---|---|---|
| `data.js` | 全部游戏内容（房间/道具/配方/岗位/彩蛋/文案/地图） | ✅ 只改这个 |
| `index.html` | 引擎（渲染/合成逻辑/图鉴/地图/编辑器） | ❌ 不用动 |

## 快速上手：三个网址

| 网址后缀 | 用途 |
|---|---|
| `index.html` | 正常游戏 |
| `index.html?edit=1` | **布局编辑模式**：所有道具（含未合成的）直接显示；拖卡片=移动，拖右下角紫色手柄=缩放拉伸（横竖独立），双击=恢复默认大小；顶部"导出数据"一键复制最新坐标和尺寸粘回 data.js |
| `index.html?debug=1` | **数据校验报告**：检查 id 重复、配方引用不存在的道具、组合冲突、坐标越界、地图漏房间等 |

## 替换要素的步骤

1. **改 `data.js`**（各字段有注释说明）：
   - `baseItems` 基础道具：`id`（唯一英文名，配方靠它引用）、`emoji`、`name`、`room`、`x/y` 坐标、可选 `w/h`（自定义宽高 px，默认 60×60，范围 36–180，emoji 随尺寸缩放）
   - `itemRecipes` 物品配方：`a + b => result`，result 也会作为新道具出现在房间里
   - `jobRecipes` 岗位配方：`a + b => 岗位`，desc 中 `\n` 换行
   - `easterEggs` 彩蛋：`[a, b, 吐槽文案]`，无产出纯搞笑
   - `genericFails` 失败文案：随机抽一条
   - `rooms` + `mapLayout` + `roomDecos`：房间、平面图布局、家具布景
2. **改 `version`**：整套内容替换时必须改（如 `v22` → `v23`），玩家旧存档自动作废，避免脏数据
3. **打开 `?debug=1`** 确认校验全绿
4. **打开 `?edit=1`** 拖道具摆位 → 导出数据 → 粘回 data.js
5. `git push` 自动部署上线

## 新增房间

```js
// rooms 里加一条（scene 先用 'plain' 灰色通用底，等有空再画专属家具）：
{id:'canteen',name:'食堂',emoji:'🍚',unlockAt:3,unlockText:'合成3个岗位解锁',scene:'plain'},
```
- `mapLayout` 里把 `'canteen'` 排进某一行（一行只放一个 = 横跨整行）
- `roomDecos` 加 `canteen:''`（家具可以以后补，class 写法参考 index.html `<style>` 里的 `.f.front-*` 等）
- 场景主题可复用现成的五套：`front`(暖木前台) / `admin`(灰蓝办公) / `lab`(深色机房) / `market`(米色创意) / `boss`(深棕豪华)

## 注意事项

- 道具 `id` 全局唯一，改 id 记得同步所有引用它的配方（`?debug=1` 会兜底查出来）
- 同一个 `a+b` 组合只能有一个产出（物品配方、岗位配方、彩蛋三者互斥）
- 坐标是场景百分比：x 建议 0–92，y 建议 0–88（卡片自身占宽高）
- 台面类道具想"放在家具上"：卡片底边 ≈ y + 7~12（视屏幕高度），拖一下最直观
