# 内容替换手册

本项目已拆分为**引擎 + 数据**两层，并采用**双版本目录**：

| 位置 | 版本 | 访问地址 |
|---|---|---|
| 仓库根目录 | **正式版**（玩家玩的） | `https://realzjr.github.io/job-craft-game/` |
| `edit/` | **编辑版**（你随便折腾） | 本地 `http://localhost:8765/edit/index.html?edit=1` |

**日常流程**：只在 `edit/` 里改内容 → 本地测试满意 → 终端跑 `./sync.sh` 一键发布（自动提交+部署）。发布前如果内容变了但 `version` 没改，脚本会提醒。想放弃编辑版的改动，跑 `./sync.sh pull` 用正式版重置。

| 文件 | 作用 | 换内容时要不要动 |
|---|---|---|
| `edit/data.js` | 全部游戏内容（房间/道具/配方/岗位/彩蛋/文案/地图） | ✅ 只改这个 |
| `edit/index.html` | 引擎（渲染/合成逻辑/图鉴/地图/编辑器） | ❌ 不用动 |
| `sync.sh` | 发布脚本：`./sync.sh` 发布（**每次自动打 git tag 留档**）/ `list` 看发版历史 / `rollback <tag>` 一键回滚正式版 / `pull` 用正式版重置编辑版 | 直接运行 |

## 快速上手：三个网址

| 网址后缀 | 用途 |
|---|---|
| `index.html` | 正常游戏 |
| `index.html?edit=1` | **布局编辑模式**：所有道具（含未合成的）直接显示；拖卡片=移动，拖右下角紫色手柄=缩放拉伸（横竖独立），双击=恢复默认大小；图鉴内可**➕添加公式**（物品配方/岗位，材料下拉选）和**✕删除公式**（有依赖会拦截提示）；顶部"导出数据"一键复制全部改动粘回 data.js |
| `index.html?debug=1` | **数据校验报告**：检查 id 重复、配方引用不存在的道具、组合冲突、坐标越界、地图漏房间等 |

## 替换要素的步骤

1. **改 `data.js`**（各字段有注释说明）：
   - `baseItems` 基础道具：`id`（唯一英文名，配方靠它引用）、`emoji`、`name`、`room`、`x/y` 坐标、可选 `w/h`（自定义宽高 px，默认 60×60，范围 36–180，emoji 随尺寸缩放）
   - `itemRecipes` 物品配方：`a + b => result`，result 也会作为新道具出现在房间里
   - `jobRecipes` 岗位配方：`a + b => 岗位`，desc 中 `\n` 换行
   - `easterEggs` 彩蛋：`[a, b, 吐槽文案]`，无产出纯搞笑
   - `genericFails` 失败文案：随机抽一条
   - `rooms` + `mapLayout` + `roomDecos`：房间、平面图布局、家具布景
2. **改 `version`**：整套内容替换时必须改（如 `v22` → `v23`），玩家旧存档自动作废，避免脏数据
3. **打开 `?debug=1`** 确认校验全绿
4. **打开 `?edit=1`** 拖道具摆位 → 导出数据 → 粘回 data.js
5. `git push` 自动部署上线

## 新增房间

```js
// rooms 里加一条（scene 先用 'plain' 灰色通用底，等有空再画专属家具）：
{id:'canteen',name:'食堂',emoji:'🍚',unlockAt:3,unlockText:'合成3个岗位解锁',scene:'plain'},
```
- `mapLayout` 里把 `'canteen'` 排进某一行（一行只放一个 = 横跨整行）
- `roomDecos` 加 `canteen:''`（家具可以以后补，class 写法参考 index.html `<style>` 里的 `.f.front-*` 等）
- 场景主题可复用现成的五套：`front`(暖木前台) / `admin`(灰蓝办公) / `lab`(深色机房) / `market`(米色创意) / `boss`(深棕豪华)

## 批量把 emoji 换成图片（最快路径）

**方式一：编辑模式里直接上传（推荐）**
1. 用 `python3 devserver.py 8765` 启动本地服务器（普通 http.server 不支持上传）
2. 打开编辑模式，每个道具卡片/图鉴条目/平面图房间左上角都有 **📷 按钮**
3. 点 📷 选图 → 自动压成 256px PNG、按约定文件名存入 `edit/assets/`、自动打开 `useImages` 开关、立即显示
4. 发布时 `./sync.sh` 会把图片一起带上线

**方式二：批量放文件**
1. 打开 `assets/MANIFEST.md`，里面是**全部需要的图片文件名清单**（道具按 id、岗位按中文名、房间按 id），直接拿去喂批量出图工具
2. 图片放进对应目录：`edit/assets/items/`、`edit/assets/jobs/`、`edit/assets/rooms/`（建议正方形透明底 PNG，256×256 足够）
3. `edit/data.js` 里 `useImages:false` 改成 `true` —— 完成，**一行数据都不用改**

**两种方式通用：**
- 缺图的条目自动回退显示 emoji，所以可以**分批替换**，随做随传
- 个别图片想用别的文件名/路径：给那条数据加 `img:'items/xxx.png'` 覆盖约定
- 图片生效范围：房间道具卡、合成槽、图鉴、合成结果弹窗、公式行、提示、房间标题、公司平面图（全部界面同步替换）
- 换了整套内容后想重新生成清单：让 Claude 重跑 manifest 生成脚本即可

## 注意事项

- 道具 `id` 全局唯一，改 id 记得同步所有引用它的配方（`?debug=1` 会兜底查出来）
- 同一个 `a+b` 组合只能有一个产出（物品配方、岗位配方、彩蛋三者互斥）
- 坐标是场景百分比：x 建议 0–92，y 建议 0–88（卡片自身占宽高）
- 台面类道具想"放在家具上"：卡片底边 ≈ y + 7~12（视屏幕高度），拖一下最直观
